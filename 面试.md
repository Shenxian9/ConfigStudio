# 项目定位
这是一个 **Qt Widgets + Qwt 的工业 HMI 组态编辑与运行预览工具**，面向嵌入式/触控/Wayland 场景，核心目标是把“界面设计、属性配置、变量绑定、运行态显示”串成一体化工作流。项目强调设计态/运行态分离、组件属性统一接口、以及变量驱动的绑定分发。技术栈主要是 C++17、Qt（Widgets/Quick/QML/SerialPort）、Qwt、qmake。近几次提交明显聚焦在 **数据源接入（串口）+ 映射到 VariableModel + Wayland/IME 稳定性 + 测试补齐**。最可能被老师深挖的方向是：**Qt 信号槽数据流、模型设计、串口帧解析、UI 生命周期与输入法问题、工程化与可测性**。

# 2分钟项目介绍稿
老师好，我介绍一下我的项目 ConfigStudio。它是一个运行在嵌入式板卡上的 Qt Widgets 工业 HMI 组态与运行预览工具。项目目标是让用户在设计态完成控件拖拽、属性配置和变量绑定，在运行态稳定展示业务数据。  
架构上我把系统分成 UI、Canvas、Components、Runtime、Model、Datasrc 六层，核心是 VariableModel 与 DataBindingManager。外部数据（例如串口）不会直接改控件，而是先更新模型，再由绑定管理器分发到组件属性，这样一致性和可维护性更好。  
我参与的重点是数据源闭环和稳定性：在 DataWorkspace 里实现了串口配置、启停、key 到 varId 映射、状态树展示，形成 SerialDataSource -> SerialVariableMapper -> VariableModel 的完整路径。为适配现场输入，解析层支持 key=value 多字段，并兼容分号和逗号分隔。  
项目难点是 Wayland + 触控 + 输入法的生命周期问题。我们采用嵌入式编辑面板、输入法状态治理、延迟焦点切换和异步刷新，降低了崩溃风险。  
工程化方面我补了两类单测：VariableModel 行为测试和 SerialMapper 解析映射测试。后续我计划继续做多协议数据源抽象、映射持久化和自动化 UI 压测。

# 30秒项目介绍稿
这是一个 Qt Widgets 工业 HMI 组态工具，核心是“模型驱动绑定”：外部串口数据先映射到 VariableModel，再由 DataBindingManager 分发到控件属性。我主要做了 DataWorkspace 的串口配置、映射、启停和状态展示闭环，并处理了 Wayland+触控+IME 的稳定性问题，同时补充了模型层和解析层单测。
