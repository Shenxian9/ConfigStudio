# ConfigStudio

> 基于 **Qt Widgets + Qwt** 的工业 HMI 组态编辑与运行预览工具。

ConfigStudio 面向工业监控、人机界面（HMI）和嵌入式可视化场景，提供从界面设计、属性配置、变量绑定到运行态预览的一体化工作流。

## 1. 项目概览

ConfigStudio 的目标是：

1. 用可视化方式快速搭建工业监控页面；
2. 将控件属性与变量数据源解耦并绑定；
3. 在运行态以低误操作风险展示与交互；
4. 支持触屏设备和嵌入式场景中的编辑体验。

**定位特点**：

- 设计态（Design Mode）：布局、拖拽、缩放、属性编辑；
- 运行态（Runtime Mode）：只读预览为主，保留必要业务交互；
- 组件化架构：每个组件独立暴露 `properties()` / `setPropertyValue()`；
- 绑定驱动：通过 `varId` 将变量更新分发给控件。

## 2. 核心能力

### 2.1 画布设计

- 拖拽式添加控件；
- 组件选中、移动、缩放；
- 属性表实时反映选中组件属性；
- 属性编辑支持快速点击切换与文本输入。

### 2.2 组件体系

当前项目内典型组件包括（不限于）：

- `plot`（曲线图，支持多曲线绑定）
- `indicator`（状态灯，支持闪烁逻辑）
- `text`（文本显示）
- `numeric`（数值显示）
- `dial / thermo / histogram / slider / wheel / switch`

### 2.3 绑定与仿真

- `DataBindingManager` 将变量变化分发到组件属性；
- 组件通过 `varId`（或 `varId1..N`）接收数据；
- `RuntimeSimulator` 支持本地仿真，便于无 PLC 环境下联调。

### 2.4 触屏/嵌入式友好

- 常见属性支持“点按切换”而非键盘输入；
- 属性编辑采用内嵌输入面板，降低 Wayland/IME 焦点切换风险；
- 表格行高、字体、滚动行为针对触摸场景优化。

## 3. 系统架构与数据流

### 3.1 分层结构

- **UI 层**：`ui/`（主界面、属性表、交互逻辑）
- **画布层**：`canvas/`（CanvasView / CanvasItem）
- **组件层**：`components/`（各控件实现）
- **运行层**：`runtime/`（运行窗口、绑定管理、仿真）
- **模型层**：`model/`（变量模型）

### 3.2 典型数据流

1. 用户在画布选择一个组件；
2. `MainWindow` 调用该组件 `properties()` 生成属性表；
3. 用户修改属性（点击切换或输入）；
4. `MainWindow` 调用组件 `setPropertyValue(key, value)`；
5. 若是绑定属性（如 `varId`），运行层将变量更新推送到组件；
6. 组件更新自身状态并重绘。

## 4. 关键模块说明

## 4.1 `canvas/`

- `CanvasItem`：所有可视控件基类，统一行为接口；
- `CanvasView`：承载控件实例，管理选中、拖拽、缩放、删除等。

**关键点**：

- `setEditLocked(bool)` 用于运行态锁定编辑行为；
- 设计态允许布局操作，运行态阻断误拖拽/误缩放。

## 4.2 `components/`

每个组件遵循统一接口：

- `type()`：组件类型标识；
- `properties()`：暴露可编辑属性给属性面板；
- `setPropertyValue()`：接收属性回写并更新控件状态。

例如：

- `PlotComponent`：支持 `curveCount`、`varId1..N`、历史表格弹层；
- `IndicatorComponent`：支持 `on / blink / mode / onColor / offColor / Interval/Ms`；
- `TextComponent`：支持 `text / textColor / font / align / fontSize`。

## 4.3 `runtime/`

- `RuntimeWindow`：运行态容器窗口；
- `DataBindingManager`：变量到组件属性的路由器；
- `RuntimeSimulator`：模拟变量流。

## 4.4 `ui/`

`MainWindow` 是核心交互协调者：

- 组织组件面板与画布；
- 属性表渲染与编辑；
- 运行态切换；
- 触屏输入与弹层管理。

---

## 5. 组件属性机制（重点）

本项目的属性编辑机制强调“统一接口 + 组件自治”。

- 属性表展示：由组件 `properties()` 决定；
- 属性回写：统一进入组件 `setPropertyValue()`；
- 组件内部负责：类型解析、容错、重绘触发、绑定处理。

### 5.1 快速切换属性

以下类型通常支持点按切换：

- 布尔属性（如 `on`, `blink`）；
- 枚举属性（如 `mode`, `align`, `font`）；
- 颜色属性（如 `onColor`, `offColor`, `textColor`）。

### 5.2 文本输入属性

- 非切换类属性走内嵌输入面板；
- 编辑完成后写回组件，再刷新属性表；
- 该机制可减少 Wayland 下 IME 生命周期冲突。

## 6. 运行态与设计态机制

### 6.1 设计态

- 允许控件移动、缩放、删除；
- 属性可编辑；
- 可进行绑定配置。

### 6.2 运行态

- 通过 edit-lock 锁定编辑行为；
- 重点是观察与业务交互（如开关、滑块等）；
- 避免误改布局。

### 6.3 安全性策略

- 进入运行态前清理选择态；
- 锁定画布组件编辑；
- 退出运行态恢复设计能力。

## 8. 开发与调试建议

### 8.1 启用属性/组件诊断日志

```bash
CONFIGSTUDIO_PROP_DIAG=1 ./ConfigStudio
```

日志可帮助定位：

- 属性表点击是否进入正确分支；
- setPropertyValue 是否收到预期 key/value；
- 绑定更新是否到达目标组件。

### 8.2 Wayland/IME 场景建议

- 避免频繁使用顶层模态对话框；
- 优先使用嵌入式输入面板或内嵌提示层；
- 尽量避免焦点切换时重建复杂 UI。

### 8.3 组件属性设计建议

新增组件时建议：

1. 属性命名明确（避免与其他控件歧义）；
2. `properties()` 与 `setPropertyValue()` 一一对应；
3. 关键属性具备容错解析；
4. 更新后立即触发可见反馈（重绘/刷新）。

## 9. 典型使用流程

1. 打开 ConfigStudio，进入设计态；
2. 拖拽 `text / indicator / plot` 等控件到画布；
3. 在属性表配置文本、颜色、模式、阈值、`varId`；
4. 对 `plot` 可设置 `curveCount` 并配置 `varId1..N`；
5. 切换运行态，观察实时数据显示；
6. 双击 `plot` 查看历史表格，辅助调试。

## 10. 常见问题（FAQ）

### Q1: 属性点击后为什么有的会弹输入框？

因为只有“可切换型属性”走点击切换；自由文本/数值属性走输入面板。

### Q2: 多曲线 plot 为什么限制重复 varId？

这是为了避免同一变量重复绑定导致曲线语义混乱。冲突会提示并拒绝绑定。

### Q3: 运行态为什么不能拖动控件？

运行态默认锁定编辑，保证预览稳定并避免误触改布局。

### Q4: 中文文本出现边界问题如何处理？

`TextComponent` 已按 `QFontMetrics` 做拟合缩放；建议同时避免极端长文本与过窄控件宽度。
